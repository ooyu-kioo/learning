## 値オブジェクトを使うメリット
- 表現力が増す
  - その値が何を表すのかの自己ドキュメント性が高い
- 不正な値を弾く
  - validation系
- 誤った代入を防ぐ
- ロジックの散財を防ぐ


## 値オブジェクトの性質

### 不変である
- 謎の状態変更によるバグを防ぐ
- (メモリ消費削減とかで)オブジェクトをcashして使いまわせる

## 交換が可能である(再インスタンス化・再代入によって)

## 等価性によって比較される
振る舞いとして比較を持つことで、比較の振る舞いが集約できる

こんな感じでmethodを用意してあげて、引数はobjで受け取る
```rb
class value_obj
  # ...
  def ==(other)
    self.class == other.class &&
      length == other.length &&
      unit == other.unit
  end
  alias eql? ==
  # ...
end
```
